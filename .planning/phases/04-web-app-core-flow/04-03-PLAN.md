Create `apps/web/src/components/docpipe/docpipe-workspace.tsx` as the main `'use client'` orchestration component. It must manage these exact pieces of state:
- `selectedFile: File | null`
- `selectedTemplateId: 'invoice' | 'receipt' | 'w2' | ''`
- `apiKey: string`
- `isExtracting: boolean`
- `errorMessage: string | null`
- `resultJson: string | null`
- `overallConfidence: number | null`

Implement upload behavior through `upload-dropzone.tsx`:
- visible drag target text includes `PDF, PNG, or JPG`
- supports click-to-select with a hidden `<input type="file">`
- `accept` attribute must be `application/pdf,image/png,image/jpeg`
- on drop or input change, reject unsupported file types before extraction and set `errorMessage` to `That file is not supported. Use PDF, PNG, or JPG and try again.`

Implement `api-key-field.tsx`:
- label text `Anthropic API key`
- helper copy `Stored only for this browser session`
- value is driven by `useSessionStorageState('docpipe.anthropicApiKey', '')`

Implement `template-selector.tsx`:
- list exactly three options from `BUILT_IN_TEMPLATES`
- label text `Template`

Implement extraction in `docpipe-workspace.tsx` using this concrete flow:
1. Validate `selectedFile`, `apiKey`, and `selectedTemplateId` before running.
2. Convert the file to base64 via `readFileAsBase64(selectedFile)`.
3. Resolve the selected template from `BUILT_IN_TEMPLATES`.
4. Build the Anthropic model with `createAnthropicProvider({ apiKey })`.
5. Call `extract()` with the selected schema and schema metadata.
6. Serialize `result.data` with `JSON.stringify(result.data, null, 2)` into `resultJson`.
7. Save `result.overallConfidence` into state.

Do not call any DocPipe API route for extraction. Do not use `fetch('/api/extract')`. The extraction handler must run directly in the browser via the shared core package.

Implement `results-preview.tsx` so the right-side panel renders:
- heading `Result preview`
- `overallConfidence` shown as a percentage if available
- a `<pre>` block with formatted JSON when extraction succeeds
- placeholder copy before first run

Update `apps/web/src/app/page.tsx` so the server component only renders the page frame and the client `DocpipeWorkspace`.
